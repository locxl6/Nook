## 引言

随着大语言模型（LLM）技术的快速发展，AI 聊天助手已成为日常工作和学习的重要工具。然而，现有的 AI 聊天产品大多依赖云端服务，存在数据隐私泄露风险，且需要持续联网才能使用。Ollama 等本地模型部署工具的出现，使得在个人电脑上运行大模型成为可能。本项目旨在开发一款基于本地大模型的 AI 聊天桌面应用 Nook，让用户无需联网即可享受 AI 聊天服务，同时保障数据隐私。

## 任务概述

### 1、目标

实现一个基于本地大模型的 AI 聊天桌面应用，支持多会话管理、SSE 流式响应、Markdown 实时渲染等功能，所有数据本地存储，无需联网，保障用户隐私。

### 2、特点

- **完全本地化：** 模型推理、数据存储均在本地完成，无需联网。
- **隐私保护：** 用户对话数据不会上传到云端。
- **流式响应：** 采用 SSE 技术实现逐 token 输出，用户体验流畅。
- **跨平台：** 基于 Electron 构建，支持 Windows、macOS、Linux。
- **Markdown 渲染：** 支持代码高亮、数学公式、表格等富文本格式。

## 需求分析

### 1、功能分析

#### 1.1、聊天功能

- 用户可以创建新的聊天会话，在会话中发送消息，AI 以 SSE 流式方式逐 token 返回响应，支持停止生成和重新生成。
- 支持 Markdown 格式渲染，包括代码块语法高亮、表格、列表、LaTeX 数学公式（行内公式用 `$` 包裹，块级公式用 `$$` 包裹）。
- 支持多轮对话，系统自动保留最近 20 条历史消息作为上下文，确保对话连贯性。
- 支持消息复制、删除单条消息等操作。

#### 1.2、会话管理

- 用户可以创建新会话，系统自动以用户首条消息前 20 字符作为默认标题，也支持手动修改会话标题。
- 侧边栏展示所有会话列表，按最近更新时间排序，支持切换会话（自动加载历史消息）、删除会话、清空会话消息。
- 支持按关键词搜索会话（包括标题模糊搜索和消息内容全文搜索）。

#### 1.3、模型管理

- 系统通过 Ollama API 获取本地已安装的模型列表，用户可以在聊天时选择使用的模型。
- Ollama 未运行时返回默认模型信息，前端显示提示。

#### 1.4、界面交互

- 采用左右分栏布局：左侧为会话列表侧边栏，右侧为聊天主区域。顶栏显示当前会话标题和主题切换按钮。

#### 1.5、主题与设置

- 支持深色/浅色主题切换，主题偏好保存在 localStorage 中，下次启动自动恢复。
- 侧边栏在窄窗口下自动折叠，支持手动展开/折叠。
- 输入框支持自适应高度，Enter 发送、Shift+Enter 换行。

### 2、用户分析

本项目面向需要 AI 聊天辅助但关注数据隐私的用户群体，主要包括：

- **程序员和开发者：** 需要 AI 辅助编程、代码解释，但不希望代码上传到云端。
- **学生和研究人员：** 需要 AI 辅助学习和写作，关注隐私。
- **对数据隐私敏感的用户：** 不希望对话内容被云端服务收集。
- **网络环境不稳定地区的用户：** 无需持续联网即可使用 AI 聊天功能。

## 附录

### 技术选型

**前端：** Electron 43 + React 19 + TypeScript + Vite 6 + Ant Design 6 + Zustand 5 + react-markdown 10

**后端：** Python 3 + FastAPI + Uvicorn + SQLite3（标准库）+ Ollama SDK

**通信：** REST API（JSON）+ SSE 流式传输（text/event-stream）

**部署：** Ollama 本地运行（默认端口 11434），后端 FastAPI 运行在 localhost:8000，前端 Electron 渲染进程直接 fetch 后端 API。

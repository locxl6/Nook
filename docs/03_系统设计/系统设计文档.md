# 概要设计文档

## 一、UI设计

系统采用左右分栏布局，整体风格参考主流 AI 聊天产品（如 ChatGPT、DeepSeek），使用 Ant Design 组件库构建。

### 1、侧边栏（Sidebar）

顶部显示 Nook Logo 和新建对话按钮。会话列表按最近更新时间倒序排列，每项显示会话标题和更新日期，鼠标悬停显示删除按钮（Popconfirm 确认删除）。窄窗口（<768px）下侧边栏自动折叠，支持手动切换。底部显示版本号。

### 2、顶栏（Header）

左侧为侧边栏折叠/展开按钮，中间显示当前会话标题（无会话时显示"新对话"），右侧为深色/浅色主题切换按钮。

### 3、聊天主区域（Content）

消息列表：用户消息右对齐（主题色背景），AI 消息左对齐（表面色背景+边框），AI 消息支持 Markdown 渲染。消息头像区分用户（UserOutlined）和 AI（RobotOutlined）。AI 流式响应时显示脉冲动画指示器。空状态显示欢迎图标和提示文案。最大宽度 768px 居中。

### 4、输入区域（ChatInput）

TextArea 自适应高度（1-6行），Enter 发送、Shift+Enter 换行。发送按钮在输入为空或正在生成时禁用。正在生成时显示 loading 状态。

### 5、消息操作

AI 消息悬停显示操作按钮：复制（复制到剪贴板，2秒反馈）、重新生成（重新调用模型生成回复）、删除（删除该消息）。

### 6、主题系统

支持深色/浅色主题切换，使用 CSS 变量（`--ds-bg`, `--ds-text-primary`, `--ds-border` 等）实现主题配色。主题偏好存储在 localStorage（key: `nook-theme`），启动时自动恢复。深色模式主色 `#3b82f6`，浅色模式主色 `#39C5BB`。

## 二、数据库设计

数据库使用 SQLite（Python 标准库 sqlite3），文件路径 `data/chatbot.sqlite3`。启用 WAL 模式和外键约束。包含两张表：

### conversations 表（会话表）

| 序号 | 字段名 | 类型 | 说明 | 备注 |
|---|---|---|---|---|
| 1 | id | TEXT | 会话唯一标识（UUID） | 主键 |
| 2 | title | TEXT | 会话标题 | NOT NULL |
| 3 | created_at | TEXT | 创建时间（ISO 8601） | NOT NULL |
| 4 | updated_at | TEXT | 更新时间（ISO 8601） | NOT NULL |

### messages 表（消息表）

| 字段名 | 类型 | 说明 |
|---|---|---|
| id | TEXT | 消息唯一标识（UUID），主键 |
| conversation_id | TEXT | 所属会话 ID，外键 → conversations.id，ON DELETE CASCADE |
| role | TEXT | 消息角色（user/assistant），NOT NULL |
| content | TEXT | 消息内容，NOT NULL |
| created_at | TEXT | 创建时间（ISO 8601），NOT NULL |

### ER 关系

```
┌──────────────┐       ┌──────────────┐
│ conversations│       │   messages   │
├──────────────┤       ├──────────────┤
│ id (PK)      │──┐    │ id (PK)      │
│ title        │  │    │ conversation │
│ created_at   │  └───>│ _id (FK)     │
│ updated_at   │       │ role         │
└──────────────┘       │ content      │
                       │ created_at   │
                       └──────────────┘
```

## 三、系统架构

系统采用前后端分离架构，后端提供 REST API + SSE 流式响应，前端为 Electron 桌面应用。

### 1、后端架构（Python + FastAPI）

| 模块 | 说明 |
|---|---|
| `main.py` | FastAPI 应用入口，注册路由和 CORS 中间件，启动时初始化数据库 |
| `routers/health.py` | 健康检查接口 `GET /health` |
| `routers/models.py` | 模型列表接口 `GET /api/models`，调用 Ollama 获取本地可用模型 |
| `routers/conversations.py` | 会话管理接口（创建、查询、删除、清空消息、修改标题、搜索） |
| `routers/chat.py` | 流式聊天接口 `POST /api/chat/stream`，使用 SSE 返回逐 token 响应 |
| `db.py` | SQLite 数据库操作层，使用原始 SQL（无 ORM），提供会话和消息的 CRUD 函数 |
| `llm.py` | Ollama 模型调用层，提供流式推理（`stream_ollama`）、系统提示词组装（`build_system_prompt`）、消息列表构建（`build_messages`，取最近 20 条历史） |
| `schemas.py` | Pydantic 数据模型，定义请求/响应结构（ChatRequest、ConversationInfo、ModelInfo、SSEChatData 等） |
| `config.py` | 全局配置（默认模型名 qwen2.5:1.5b） |

### API 接口列表

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/health` | 健康检查 |
| GET | `/api/models` | 获取可用模型列表 |
| POST | `/api/conversations` | 创建新会话 |
| GET | `/api/conversations` | 获取所有会话列表 |
| GET | `/api/conversations/{id}` | 获取会话详情 |
| DELETE | `/api/conversations/{id}` | 删除会话 |
| GET | `/api/conversations/{id}/messages` | 获取会话消息 |
| DELETE | `/api/conversations/{id}/messages` | 清空会话消息 |
| DELETE | `/api/conversations/{id}/messages/{msg_id}` | 删除单条消息 |
| PUT | `/api/conversations/{id}/title` | 修改会话标题 |
| POST | `/api/chat/stream` | SSE 流式聊天 |

### 核心数据流

```
Client (SSE) → POST /api/chat/stream → chat.py
    → db.py (查会话、取历史)
    → llm.py (build_system_prompt → build_messages → stream_ollama)
    → Ollama Server (localhost:11434)
    → StreamingResponse (text/event-stream)
    → db.py (存储 user + assistant 消息)
```

### 2、前端架构（Electron + React + TypeScript）

| 模块 | 说明 |
|---|---|
| `electron/main.ts` | Electron 主进程，创建 BrowserWindow（1200x800），加载渲染进程 |
| `electron/preload.ts` | 预加载脚本，通过 contextBridge 暴露 IPC 接口 |
| `App.tsx` | 主布局组件，组合侧边栏、聊天区、输入框，管理主题切换 |
| `hooks/useStreamChat.ts` | SSE 流式消费者，使用 fetch + ReadableStream 消费流式响应，支持 AbortController 中断生成 |
| `stores/chatStore.ts` | 聊天状态管理（消息列表、流式状态） |
| `stores/conversationStore.ts` | 会话状态管理（列表、当前选中、消息缓存，持久化到 localStorage） |
| `stores/settingsStore.ts` | 设置状态管理（后端地址、模型配置） |
| `components/Sidebar.tsx` | 侧边栏组件（会话列表、新建对话、删除确认） |
| `components/ChatInput.tsx` | 输入框组件（自适应高度、快捷键） |
| `components/ChatMessage.tsx` | 消息气泡组件（Markdown 渲染、操作按钮） |
| `components/ModelSelector.tsx` | 模型选择器组件 |
| `types/index.ts` | TypeScript 类型定义 |

### 前端状态管理

使用 Zustand 进行状态管理，三个独立 Store：

- **chatStore：** 管理当前消息列表、输入内容、流式加载状态
- **conversationStore：** 管理会话列表、当前选中会话，使用 `persist` 中间件将消息缓存到 localStorage
- **settingsStore：** 管理后端 API 地址和默认模型配置
